12.1 — Filosofía

Inversión cromática intencional

En light, el purple es el protagonista (color de marca primario sobre fondo claro). En dark, el lime toma el liderazgo porque destaca mucho mejor sobre negro y mantiene el "feeling IA" característico de QUANTIA. Los hovers también se invierten: el botón primary en dark se ilumina con lime claro, y el accent vuelve al purple corporativo en hover.

12.2 — Implementación

Estructura del index.css (Tailwind v4)

En Tailwind v4 ya no existen tailwind.config.js ni darkMode: 'class'. La configuración vive en CSS. El patrón estándar tiene tres bloques: @custom-variant para activar la clase .dark, :root y .dark para los valores que cambian, y @theme inline como puente para que Tailwind genere las utilities (bg-surface, text-primary, bg-brand...). Un solo nombre de variable por token — el modo dark lo sobreescribe.

12.3 — Carga de fuentes

Cómo importar Nunito según el framework

Importar las fuentes con @import url(...) dentro del CSS funciona, pero bloquea el render hasta que el navegador descargue y parsee el CSS antes de pedir las fuentes. La recomendación estándar es cargarlas desde el HTML/layout para permitir preconnect y descarga en paralelo. Quita el @import de Google Fonts del index.css si usas alguna de estas dos opciones.

⚡ Vite · CRA · HTML

Tag <link> en el <head> con preconnect

Coloca los preconnect y el stylesheet en index.html. El navegador abre la conexión TLS antes de necesitar la fuente y descarga en paralelo con el resto de assets.

<!-- index.html --> <head> <link rel="preconnect" href="https://fonts.googleapis.com"> <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin> <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Nunito:wght@600;700;800;900&family=Nunito+Sans:wght@400;500;600;700&display=swap" > </head>

En index.css las fuentes ya están disponibles globalmente. No hace falta el @import url(...). El @theme sigue declarando --font-display y --font-sans como antes.

▲ Next.js (App Router)

next/font/google · auto-hostado y sin layout shift

Next descarga las fuentes en build time, las sirve desde tu propio dominio (cero requests a Google en producción) y previene el flash automáticamente. Es la opción recomendada por el equipo de Next.

// app/layout.jsx import { Nunito, Nunito_Sans } from "next/font/google"; const nunito = Nunito({ subsets: ["latin"], weight: ["600","700","800","900"], variable: "--font-display", }); const nunitoSans = Nunito_Sans({ subsets: ["latin"], weight: ["400","500","600","700"], variable: "--font-sans", }); export default function RootLayout({ children }) { return ( <html lang="es" className={`${nunito.variable} ${nunitoSans.variable}`}> <body>{children}</body> </html> ); }

Next inyecta --font-display y --font-sans como CSS vars en el <html>. En @theme no hace falta declararlas, ya las hereda. Quita tanto el @import url(...) como las declaraciones --font-* del @theme si vas por esta vía.
